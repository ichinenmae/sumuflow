# 開発ガイドライン

## 基本方針

シンプルさを最優先とする。

大規模Webアプリ化を目的としない。

個人利用ツールとして保守しやすい構造を重視する。

---

## 使用技術

- HTML
- CSS
- Vanilla JavaScript

---

## 使用禁止

- React
- TypeScript
- Redux
- Next.js
- Vite
- 複雑なビルド環境

---

## 推奨構成

/
├─ index.html
├─ style.css
├─ app.js
├─ manifest.json
├─ README.md
├─ CHANGELOG.md
└─ icon/

---

## バージョニング

セマンティックバージョニングを採用する。

形式

MAJOR.MINOR.PATCH

例

1.0.0
1.1.0
1.1.1

ルール

MAJOR
互換性破壊

MINOR
新機能追加

PATCH
バグ修正

---

## ソースコードコメント

主要ファイル先頭に記載すること。

例

/*
App Name: Delivery Cashflow Dashboard
Version: 1.0.0
Generated By: Claude Code
Updated: 2026-05-26
*/

---

## Generated By

実際にコード生成を行ったAI名を記録すること。

例

- Claude Code
- Codex
- Cursor
- Cline
- Windsurf
- ChatGPT

---

## CHANGELOG管理

CHANGELOG.md を作成し、
バージョンごとの変更内容を記録すること。

---

## Storage設計

StorageServiceを作成すること。

UIから直接LocalStorageへアクセスしないこと。

---

## 可読性

短く理解しやすいコードを優先する。

過度な抽象化は禁止。

---

## README必須

README.md を作成すること。

記載内容

- GitHub Pages公開方法
- JSONバックアップ方法
- JSON復元方法
- バージョニングルール

---

## テスト

最低限確認すること。

- 支払い0件
- 入金0件
- 期限超過あり
- 期限超過なし
- 不足額あり
- 不足額なし
- JSON Export
- JSON Import
- 支払い複製
- 入金複製
- 残高未更新警告
