# Delivery Cashflow Dashboard

配達員向けの資金ショート予測ダッシュボードです。現在残高、支払い、入金予定から未来の残高をシミュレーションし、次回報酬までに必要な追加売上額、必要稼働時間、1日あたり必要売上を表示します。

## 技術

- HTML
- CSS
- Vanilla JavaScript
- LocalStorage

ビルド工程は不要です。`index.html` をブラウザで開くだけで動作します。

## 使い方

1. ホームで現在残高を更新します。
2. 管理画面で支払いと入金予定を登録します。
3. ホーム画面でも支払い追加、入金追加、支払い済み切替ができます。
4. 設定画面で平均売上/時間、平均件数/時間、1日の最大稼働時間、週の最大稼働日数、注意判定閾値、データ保存アラート頻度を調整します。
6. ホームで期限超過、要注意支払い、未来残高予測を確認します。

## 稼働計画

ホーム画面では、未来残高予測に加えて以下を確認できます。

- 30日単純計画: 今後30日間の支払い合計から、1日あたりの必要売上・必要稼働・件数換算を表示します。
- 30日単純計画: 週の最大稼働日数から30日間の想定稼働日数を計算し、1稼働日あたりの必要売上・必要稼働・件数換算を表示します。
- 週別必要稼働: 将来の資金ショートを避けるため、必要売上を支払日週から前週へ割り当て、週合計と1稼働日あたりの指標を表示します。
- 予想売上: 設定画面で、登録した稼働条件に基づく1日あたり・週あたりの売上見込みを表示します。
- 入力日付の注意: 土日祝など銀行休業日の前後は、実際の入金日・支払日を確認してください。

## GitHub Pages 公開方法

このフォルダの内容をリポジトリのルートに配置し、GitHub Pages の公開元をそのブランチに設定してください。サーバーやビルドツールは不要です。

## データ保存

設定画面の `データ保存` で全データを JSON ファイルとして保存できます。

## データ読込

設定画面の `データ読込` から、保存済みの JSON ファイルを選択すると復元できます。

インポート時に以下を選択できます。

- 完全上書き: 現在の全データを削除し、JSONの内容に置き換えます。
- 追記: 現在の残高と設定は維持し、JSON内の支払い・入金予定だけを新しいIDで追加します。

## データ保存アラート

設定画面でデータ保存を促すアラート頻度を変更できます。

- 初期値: 7日
- 0日: アラートなし
- 最大値: 90日

## バージョニング

セマンティックバージョニングを採用します。

- MAJOR: 互換性破壊
- MINOR: 新機能追加
- PATCH: バグ修正
